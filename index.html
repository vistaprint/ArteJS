Overview

A powerful, extensible, flexible, cross-browser rich text editor.

ArteJS is an integral part of VistaPrint's DIY product design experience that allows adding rich text to a document that eventually gets printed. This imposes a stringent requirement on the output of the rich text editor. Most of the rich text editors we evaluated either did not allow any control on the resultant markup or were simply buggy/incomplete.

In addition, ArteJS provide many other features that are expected of a modern rich text editor library.

  • A simple and intuitive jQuery like API.
    For example: 
    $(".editor").Arte(); // Turns .editor into rich text editor
$(".editor").bold(); // Make the selected text bold
$(".editor").fontSize(10); // Change fontsize of the selected text to 10px
  • A small dependency footprint: jQuery and Rangy 1.2.3 (Included with ArteJS).
  • Full control over the resultant HTML : TODO: Links to examples with Classes/Styles/Tags for a command
  • A loosely coupled architecture based on plugins that can be used to overwrite existing or add new functionality. In fact, a large part of the functionality is built using plugins.
  • ArteJS supports a clean interface for toolbar integration. TODO: Link to reference toolbar implementations that can support multiple editors with a single toolbar or one toolbar for each editor.
  • Semantically concise and valid markup: e.g. no redundant styles, classes or tags.
  • A rich event model provides a rich integration point.
  • Keyboard shortcut integration
  • State detection so that it's easy to figure out the state of selected HTML.
  • Highly granular configuration.
  • Copy/paste functionality from Microsoft Word.
  • Open source -- ArteJS is free and available on an Apache2 license.
  • Most importantly, ArteJS is used in production millions of times a day.

Typical page structure

        <!-- Markup to host the toolbar and the editor-->

        <script type="text/javascript">
            $(function () {
                // List and sequence of the toolbar buttons
                var buttons = ["bold", "italic", "underline", "blockquote", "textAlignLeft", "textAlignCenter", "textAlignRight", "h1", "h2", "h3", "h4", "h5", "h6", "subscript", "superscript", "unorderedList", "orderedList", "undo", "redo", "fontSize", "fontFamily", "color", "backgroundColor", "insertLink"];

                // Initialize the Arte editor with default options
                $(".editor").Arte();

                // Initialize the toolbar with default options
                $(".toolbar").ArteToolbar({

            });
        </script>

Rich Text API Reference

Commands

A rich text command can be applied using a html tag, a style or a class name. ArteJS allows applying the command in either of these three mechanisms.
Note that the default configuration excluded the use of deprecated html tags (e.g.: font). Configuration can be easily updated to use the deprecated html tags if needed. 
                // Calling a rich text command
                $(selector).Arte('commandName', arguments); or
                editorInstance.commandName(arguments);
Commands that ce be applied using Tag/Style/Class
  • bold
  • italic
  • underline
Commands that can be applied using Style/Class
  • align
  • backgroundColor
  • fontSize
  • fontFamily
  • color
Commands that can be applied using a html Tag
  • blockquote
  • h1-h6
  • subscript/superscript
  • ul/ol
Other commands
  • insert (Implemented using a plugin)

Events

ArteJS raises a rich collection of events to interact with the editor. 
                // Listening to the events
                $(selector).on('eventname', function(e, data) {...}); // or
                editorInstance.on('eventname', function(e, data) {...});

Events raised by editor

  • onfocus Editor gained focus.
  • onblur Editor lost focus.
  • onvaluechange Rich text value changed.
  • onmousedown Mouse down over the editor.
  • onmouseup Mouse up over the editor
  • onclick Mouse clicked over the editor.
  • onkeydown Keydown over the editor.
  • onkeypress Keypress over the editor.
  • onkeyup Key up over the editor
  • onpaste Content pasted
  • onselectionchange Selection changed.
  • onbeforecommand Before a command is executed.
  • oncommand A command has been executed.
  • oncreate An editor is created.
  • ondestroy An editor is destroyed.

Events raised by plugins

Insert/replace content Plugin
  • onbeforeinsert
  • oninsert
Paste handler
  • onbeforehandlepaste
  • onafterhandlepaste
Undo/redo
  • onbeforeundo
  • onafterundo
  • onbeforeredo
  • onafterredo

Methods

An ArteJS method or the public methods on the plugins can be called as the following: 
                // Calling a ArteJS method
                $(selector).Arte('methodName', arguments ..)
                ArteInstance.methodName(arguments)

Methods on the editor

  • value Get the innerHtml of the editor.
  • outerValue Get the outerHtml of the editor.
  • focus Set focus to the editor.
  • destroy Remove the editor from DOM.
  • on Attach an event listener to the editor.
  • off Remove the event listener from the editor.
  • toggleStyleOnElement Apply style to the editor

Public methods exposed by plugins

State Detector Plugin
  • getState Get the state of the selection for each command.
Insert plugin
  • insert Insert HTML content into the editor.
UndoManager plugin
  • hasUndo Whether the editor has undo information.
  • hasRedo Whether the editor has redo information.
  • undo Undo
  • redo Redo

ArteJS Cookbook

Editor

Creating a rich text editor

// Creating a rich text editor with default initial values ($.Arte.configuration.initialValues)
$(selector).Arte();

// Creating a rich text editor with options
var options = {
    styles: {
        "font-weight": "bold",
        "font-size": "16px"
    },
    classes: ["arte-class1", "arte-class2"],
    value: "This is new value"
};

// Construct the text area
// The following takes a set of matched elements and convert them to rich text editors
$(selector).Arte(options);

Executing Arte methods

// Once a rich text editor is created, any rich text method can now be executed as the following
$(selector).Arte("methodName", parameter1, parameter2, ...);

// Alternatively, a method can be applied to an Arte instance directly
var editorInstances = $(selector).Arte();
$(selector).Arte().each(function(){
    this.methodName(parameter1, parameter2);
});
// or
editorInstances.get(0).methodName(parameter1, parameter2);

// Example methods
$(selector).Arte("bold"); // make the selection bold
$(selector).Arte("fontSize", "12px"); // set the fontsize of the selection to 12px

// alternatively
$(selector).Arte().each(function(){
    this.bold();
    this.fontSize("12px");
});

Changing the way a rich text command is applied

// Many rich text commands can be applied using a tagName, style and class.  For example, bold command be applied as either
// style (font-weight), class (with font-weight), tags (b or strong).  Arte allows us to choose how a rich text command
// is applied using the following mechanism:
// 1) Supplied in the command options ({commandAttrType = "styleName" | "className" | "tagName"})
// 2) Inferred from the values in the options ({className = "..."})
// 3) Using default attribute type specified in the configuration file (Arte.configuration.commandAttrType)

// Arte configuration allows setting the commandAttrType globally or can be changed for each command.
// For the commands that can only be applied one way (e.g. H1, BlockQuotes, etc.) the commandAttrType is ignored.

// Bold is applied based on the global configuration
$(selector).Arte("bold");

// Override the global configuration to apply bold using B/Strong tag
$(selector).Arte("bold", { tagName: "B" });
$(selector).Arte("bold", { tagName: "STRONG" });

// Override the global configuration to apply bold using a className
$(selector).Arte("bold", { className: "arte-font-weight-bold" });

// Apply the command using the styleName
$(selector).Arte("bold", { styleName: "font-weight", styleValue: "bold" });

Getting/Setting the rich text values

// Arte allows getting the innerValue or the outerValue (like outerHTML) of the editor
// Get / set the value of a rich text editor
var value = $(selector).Arte("value");
$(selector).Arte("value", "This can be any valid html fragment.");

// Get the array of outer html of all rich text editors associated with this selector
var outerValues = $(selector).Arte("outerValue");
$(selector).Arte("value", "
This can only be a single html element 
");

Interacting with rich text command

// Suppose we want to ensure that only valid values are used in rich text commands (e.g. a whitelist
// for font-size and font-family) or we need to translate some command values (e.g. convert
// color from CMYk to RGB). This can be achieved by listening to the `onbeforecommand` event
$("#editor").Arte("on", {
    "onbeforecommand": function (e, data) {
        switch (data.commandName) {
            case "fontSize":
                switch (data.commandValue) {
                    // Translate the commandValue to a valid fontSize
                    case "Big font":
                        data.commandValue = "50px";
                        break;
                    case "Small font":
                        data.commandValue = "5px";
                        break;
                }
                break;
            case "fontFamily":
                if(!isFontInWhiteList(data.commandValue))
                {
                    // If font name is not whitelisted, don't execute the command
                    data.execute = false;
                }
                break;
        }
    }
});

Extending ArteJS

// Following is a basic template for a plugin
(function(pluginManager)
{
    // plugin internals
    var property;
    function pluginMethod() { }

    // Public methods
    $.extend($.Arte.TextArea.prototype, {
        publicMethodA: function{},
        publicMethodB: pluginMethod
    });

    var myPlugin = function ()
    {
        return {
            init: function (textArea)
            {
                // any setup or listen to textArea events
            }
        }
    }

    // Register this plugin with the plugin manager
    pluginManager.register("myPlugin", myPlugin);
})($.Arte.pluginManager)

Toolbar

Configure toolbar dropdown

// Toolbar allows setting the display value and actual value that is applied to text
// Note that the values need to be set according to the commandAttrType configuration
// Following are few of the examples of setting the styles

// Simply apply the font-size.
$.Arte.Toolbar.configuration.buttons.fontSize.options = ["", 8, 10, 12, 15, 20];

// Example where the display value different than the the actual value
$.Arte.Toolbar.configuration.buttons.fontSize.options = [
    { display: "", value: "" },
    { display: "Smaller", value: 8 },
    { display: "Small", value: 10 },
    { display: "Medium", value: 12 },
    { display: "Large", value: 15 },
    { display: "Larger", value: 20 }
];

// The actual value is name of the class to be applied
$.Arte.Toolbar.configuration.buttons.fontSize.options = [
    { display: "", value: "" },
    { display: "Smaller", value: "arte-font-weight-8" },
    { display: "Small", value: "arte-font-weight-10" },
    { display: "Medium", value: "arte-font-weight-12" },
    { display: "Large", value: "arte-font-weight-15" },
    { display: "Larger", value: "arte-font-weight-20" }
];

Compiling ArteJS

ArteJS uses Grunt to minify, combine, analyze code and run unit tests. Following are the steps to run grunt tasks:
  • Install Node
  • Install Github client
  • Create and navigate to a local directory for ArteJS code
  • Clone the git repository
  • Install node dependencies: execute "npm install"
  • Build ArteJS: Execute "Grunt all"
There are many other grunt tasks defined for conveniance.
